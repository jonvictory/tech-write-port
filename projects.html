    Work examples and projects

    2019-2020 | API Developer Documentation Alpha/Beta

    This is a mock of the API developer documentation site that I've been building. It uses a CDN called "RapiDoc" to generate the swagger UI based on json files. These json files contain the schema that make up the structure of what you see as well as the individual object descriptions. This site does not have clearance to run test calls, but with the correct credentials and server side clearance that sandbox feature would be enabled.

    I'm not a professional designer or developer, but those resources have not been made available to me, so I built the entire site by hand. I've been promised additional resources after the customer beta trials are complete. All object descriptions were created by me or created under my supervision.

    2019-2020 | API Version 11 release notes

    These are release notes for the updates that had been made to the API in between version 10 and the version 11 release during the second quarter of 2020. I managed the production of this document by the guidelines established through the Architecture and Project Management teams at Workfront.

    2019-2020 | Internal process for updating API Documentation

    This is an HTML export of a set of documentation that was originally created in Notion. It is intended as a reference for other technical writers to use when updating API documentation. The documentation assumes that technical writers are not familiar with technologies like GIT, command line OSs, and other more developer-oriented technologies that are required for these processes.

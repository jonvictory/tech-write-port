Clone the Redrock-Swagger Repository to your computer

To clone the Redrock-Swagger repository to your machine:

  1. Go to the Redrock-Swagger project overview located at: https://gitlab.workfront.tech/documentation/redrock-swagger
  1. Locate the blue button in the upper right-hand corner of the page, just below the menu bar at the top.
  1. Click the button to activate a drop down menu.
  1. Ignore the "Clone Using SSH" heading and the associated link.
  1. Locate the "Clone Using HTTPS" heading and copy the associated link.
  1. If you do not already have an instance of Ubuntu running at this time, open a new instance of Ubuntu.
  1. In the command line prompt type the following, but do not press enter: git clone
  1. Making sure there is a space after clone paste the link that you copied from the GitLab repository by:
    1. pressing "ctrl-v" (the text will not always populate automatically, if it does not, see step 2)
    1. If the copied text does not populate automatically, press the right mouse button to populate the text in the command line.
    1. The complete entry should look similar to the following: git clone https://gitlab.workfront.tech/documentation/redrock-swagger.git
  1. This command tells Ubuntu to copy the GitLab repository to your computer. It will automatically copy the repository to whatever directory that your instance currently resides in. If you have no preference (or are unsure what this means) it is fine to install the files to your default directory.
  1. Ubuntu should run a list of files and statuses as the repository is cloned to your computer, then when it is finished, you should also receive some sort of indication that the install was successful. If the installation doesn't automatically run, or there is any type of error message:
    1. Press the up arrow until the full git clone command that you just ran populates the command line.
    1. Insert sudo at the beginning of the line. The full command should resemble: sudo git clone https://gitlab.workfront.tech/documentation/redrock-swagger.git This will prompt Ubuntu to ask you for your Ubuntu credentials before cloning the repository to your machine. You will likely have to enter in your GitLab credentials as well. Note: The sudo command must be inserted when Ubuntu needs to confirm, or reconfirm, that you are in fact the "super user" associated with the current instance of Ubuntu running on your machine.

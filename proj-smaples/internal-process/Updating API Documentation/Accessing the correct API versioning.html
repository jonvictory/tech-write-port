Accessing the correct API versioning

In order to edit the correct files, you need to access the current API versioning. These files are not a part of your normal Windows OS and are stored on a partition known as the Windows Subsystem for Linux (WSL). Files stored in the WSL are not easily accessible through the channels normally available to you via the Windows OS, like Windows Explorer. In fact, it is so difficult—and potentially dangerous—to access these files that you should consider them inaccessible, no matter what you may read on the Internet. That being said, there are two main methods that you may exercise to access these files.

Accessing the correct swagger files via Ubuntu (command line)

To locate the correct swagger files via Ubuntu:

  1. Navigate to the "redrock-swagger" master folder
  1. Inside the the master folder, navigate to the sub-folder "swagger" by using the "cd" command: cd swagger
  1. Inside the swagger folder, you should see a few different directories. This structure may change over time, but generally you could see:
    1. a "template" directory.
    1. recent previous versions
    1. the current version
    1. the next version

    Note: If you don't know what the current API version is, you should not attempt to modify any files.

  1. If you comfortable using the command-line navigation, you can download extensions to open the correct files inside of Visual Studio Code using Ubuntu commands. If you are not comfortable navigating Ubuntu, you can open up the correct folder from VS Code.

Accessing the correct swagger files via VS Code (GUI)

If you do not feel comfortable installing additional programs and plugins to use in tandem with the WSL, then you can still access your local version of the Redrock Swagger Repository using Visual Studio Code.

To accesses the WSL via VS Code:

  1. Start VS Code as you would normally, in Windows—not in Ubuntu.
  1. Use the keyboard shortcut: "ctrl+shift+p"
  1. In the search field that populates, search for: "Remote-WSL: New Window"
  1. Select this option when in populates in the list of matching search terms. This will open a new VS Code window.
  1. In the new VS Code window, select "File" from the main menu (very top left-hand corner of the window), select "Open Folder" from the menu. This will open a screen that should automatically populate at the root directory of the OS you are running on WSL.
  1. If you cloned your repository to the root directory of your Ubuntu OS, then you should see the "redrock-swagger" directory displayed immediately.
  1. From the master directory, open the "swagger" folder, then select the directory that corresponds with the current API version. At the time that his documentation is being created, that directory is called: "v10.0"
  1. Opening that folder in VS Code will open all the sub-directories that you will need to update the description fields for the documentation that is generated by the Swagger UI.
